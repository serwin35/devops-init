chmod — tryb symboliczny
Składnia: chmod [kto][operator][uprawnienie] plik
Operatory
+
dodaj dodaje uprawnienie bez ruszania reszty
-
usuń usuwa uprawnienie bez ruszania reszty
=
ustaw zastępuje uprawnienia DOKŁADNIE podanymi
# dodaj execute dla właściciela
chmod u+x skrypt.sh
# usuń write dla grupy i innych
chmod go-w plik.txt
# ustaw dokładnie rw-r--r-- dla wszystkich segmentów
chmod u=rw,g=r,o=r plik.txt
# dodaj execute dla wszystkich (a = u+g+o)
chmod a+x skrypt.sh
# rekurencyjnie na cały katalog
chmod -R 755 /var/www/html
chown — zmiana właściciela
Składnia: chown [opcje] użytkownik[:grupa] plik/katalog
# zmień właściciela na www-data
chown www-data plik.php
# zmień właściciela i grupę jednocześnie
chown serwin:developers projekt/
# zmień tylko grupę (dwukropek bez usera)
chown :docker /var/run/docker.sock
# rekurencyjnie cały katalog
chown -R serwin:serwin ~/projekty/
# przenieś własność z jednego usera na drugiego
chown --from=olduser newuser -R /data
-R
rekurencyjnie — dotyczy całego drzewa katalogów
-v
verbose — wyświetla każdą zmianę
--reference=plik
kopiuje własność z innego pliku
chgrp — zmiana grupy
Dedykowane narzędzie do zmiany grupy pliku/katalogu
# zmień grupę pliku
chgrp docker /var/run/docker.sock
# rekurencyjnie cały katalog
chgrp -R www-data /var/www/
# skopiuj grupę z innego pliku
chgrp --reference=wzorzec.txt plik.txt
💡 chgrp to skrót dla chown :grupa — oba robią to samo

useradd / adduser — tworzenie użytkownika
# utwórz usera z home dir (adduser = interaktywny)
adduser serwin
# useradd = niskopoziomowy, wymaga opcji ręcznie
useradd -m -s /bin/bash -G sudo,docker serwin
-m
utwórz katalog domowy /home/user
-s /bin/bash

-G grp1,grp2
dodatkowe grupy (nie primary)
-g grupa
główna grupa (primary group)
-u 1500
ręczne UID
-d /custom/home
niestandardowy katalog domowy
-r
system account (brak home, niski UID)
-c "Opis"
komentarz / pełna nazwa (GECOS)
usermod — modyfikacja użytkownika
# dodaj usera do grupy (zachowaj stare grupy)
usermod -aG docker serwin
# dodaj do wielu grup jednocześnie
usermod -aG sudo,docker,www-data serwin
# UWAGA: -G bez -a ZASTĘPUJE wszystkie grupy!
usermod -G docker serwin  # ⚠ niebezpieczne

Specjalne bity — setuid, setgid, sticky
setuid
Wykonuje plik z uprawnieniami właściciela (nie osoby uruchamiającej)
Wyświetlany jako s w pozycji x ownera. np. /usr/bin/passwd
chmod u+s plik   # lub chmod 4755 plik
setgid
Na pliku: wykonaj jako grupa właściciela. Na katalogu: nowe pliki dziedziczą grupę katalogu
Wyświetlany jako s w pozycji x grupy
chmod g+s katalog/   # lub chmod 2755
sticky
Na katalogu: tylko właściciel pliku lub root może go usunąć, nawet jeśli ma write na katalogu
Wyświetlany jako t w pozycji x other. np. /tmp
chmod +t /shared/   # lub chmod 1777
⚠ setuid na katalogu jest ignorowany przez Linux (tylko BSD)
setuid root na skryptach też często blokowany przez bezpieczeństwo jądra
sudo i sudoers
# dodaj usera do grupy sudo (Debian/Ubuntu)
usermod -aG sudo serwin
# dodaj usera do grupy wheel (RHEL/CentOS/Fedora)
usermod -aG wheel serwin
Edytuj TYLKO przez: visudo
# /etc/sudoers — pełny sudo
serwin ALL=(ALL:ALL) ALL
# sudo bez hasła
serwin ALL=(ALL) NOPASSWD: ALL
# tylko wybrane komendy
serwin ALL=(ALL) /usr/bin/systemctl, /usr/bin/apt
# dla całej grupy (znak %)
%developers ALL=(ALL) NOPASSWD: /usr/bin/docker
Pliki drop-in (bezpieczniejsze niż edycja sudoers)
visudo -f /etc/sudoers.d/serwin
Wzorce — realne scenariusze
🌐 Serwer WWW
# ustaw nginx jako właściciela html
chown -R www-data:www-data /var/www/html
chmod -R 755 /var/www/html
# deweloper może edytować (setgid na katalogu)
usermod -aG www-data serwin
chmod g+s /var/www/html
🔑 SSH Keys
# poprawne uprawnienia dla SSH
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_rsa
chmod 644 ~/.ssh/id_rsa.pub
chmod 600 ~/.ssh/authorized_keys
🐳 Docker bez sudo
# dodaj usera do grupy docker
usermod -aG docker serwin
# aktywuj bez restartu sesji
newgrp docker
# weryfikacja
docker ps
📁 Katalog współdzielony
# katalog dla zespołu (setgid + sticky)
mkdir /shared/team
chown root:developers /shared/team
chmod 2770 /shared/team
# 2=setgid, 7=rwx owner, 7=rwx group, 0=--- other
